# Parte 5 — Power BI (pendiente)

Esta carpeta debe contener `AndinaRetail.pbix` y su exportación
`AndinaRetail_tableros.pdf`, exigidos en la Sección 6.5 del PG.

No se desarrolló en este entorno porque Power BI Desktop es una aplicación
exclusiva de Windows. Los datos ya están listos en `../datos/` (tiendas,
productos, clientes, ventas, inventario) para importarse directamente y
construir:

1. El modelo de datos (relaciones entre tablas + tabla de calendario).
2. Medidas DAX para los KPI clave (ventas, margen, ticket promedio,
   clientes activos, tasa de inactividad, etc.).
3. Al menos 3 páginas de tablero: (a) ejecutivo con KPIs; (b) ventas por
   ciudad/categoría/canal/tiempo; (c) clientes y segmentación (RFM/churn).
   Se recomienda una cuarta página con resultados predictivos/prescriptivos
   de las Partes 3 y 4.
4. Interactividad (segmentadores, drill-down/drill-through, marcadores) y
   storytelling con datos.

Esta parte se completará por separado.
